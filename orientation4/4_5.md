(W) Using the names of the simulator classes, explain the simulator a) what A-phase coding means? b) what coding B-events mean? c) what coding C events means?

You can also write Java or pseudocode.

Present a general solution. No need for the target system concepts.


a). A-phase coding means writing the part of the simulator that controls time progression.
```java
Event nextEvent = eventList.removeNextEvent();
Clock.getInstance().setTime(nextEvent.getEventTime());
```
The A-phase doesn't change the system itself. Its main job is to advance time to the next important moment.

b). Coding B-events means writing the actions for the events taht are already scheduled to happen at a specific time.
example events:
- Customer arrives
- Service ends
- Customer exits system

example pseudocode:
```java
if (event.getType() == EventType.ARRIVAL) {
    Customer customer = new Customer();
    servicePoint.addToQueue(customer);

    arrivalProcess.generateNextEvent(eventList);
}

if (event.getType() == EventType.EXIT) {
    servicePoint.setAvailable(true);
    completedCustomers++;
}
```
B-events are executed when the simulation clock reaches their scheduled time.

c). Coding C-events means writing the logic for conditional events.
A C-event is not scheduled for a fixed time like B-event. It happens only if the correct conditions are true.

Example condition:
- A customer is waiting in the queue **AND** the service point is free

example pseudocode:
```java
if (!servicePoint.isQueueEmpty() && servicePoint.isAvailable()) {
    Customer customer = servicePoint.removeFromQueue();
    servicePoint.setAvailable(false);

    double serviceTime = serviceTimeGenerator.sample();
    double exitTime = Clock.getInstance().getTime() + serviceTime;

    eventList.addEvent(new Event(EventType.EXIT, exitTime));
}
```
C-events are checked repeatedly after B-events until no more conditional events can happen.
